Lotus

Lotus is a personal-finance web app that turns raw bank and credit-card exports into a clear picture of where money actually goes — built as a fully serverless, single-author full-stack project. It's live at lotus-finance.me.

Architecture

  • Frontend: React + Vite + TypeScript, with Recharts for visualizations, PapaParse for CSV, and pdf.js for pay-stub parsing — deployed as a static site on GitHub Pages.
  • Backend: a zero-dependency Cloudflare Worker API using only the Web Crypto API.
  • Storage: Cloudflare KV for transactions, income, and auth.
  • CI/CD: GitHub Actions builds the frontend and redeploys the Worker on every push.

Features

  • Secure login — password plus TOTP two-factor auth (PBKDF2 + HMAC-SHA1), with first-run setup and QR provisioning.
  • Data entry — upload Chase / credit-card CSVs, manually log expenses, or import pay stubs (PDF or CSV) with auto-parsed tax deductions.
  • Dashboard — spending-trend line graphs, a monthly expense table, income-vs-expenditure comparison, and average monthly spend per category.
  • Smart categorization — transactions auto-sorted into ~17 categories, with credit-card payments automatically skipped.

Engineering highlights

  • Implemented 2FA cryptography from primitives (PBKDF2, HMAC-SHA1) on the Web Crypto API — no auth libraries.
  • Kept the Worker dependency-free for a tiny, fast, auditable edge backend.
  • Origin-locked CORS and a JWT secret managed through Wrangler secrets.
